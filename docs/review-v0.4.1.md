# Living Worlds v0.4.1 批判性评审

**日期** 2026-07-20
**对象** 《Living Worlds：AI 生命世界产品与系统设计文档》v0.4.1 草案
**结论** 文档的自我批判质量很高，v0.4/v0.4.1 修掉的问题（Elo、预算基因化、注意力直驱进化、孵化成本归属）都是真问题。剩余问题集中在几处"两个各自正确的补丁叠加后互相矛盾"的地方，以及对玩家真正消费的叙事层投入的设计密度远低于可验证的模拟层。

---

## 一、结构性问题（会改变产品成立性判断）

### R1. 节奏两难：牵挂要慢时间，进化要快时间

模拟日/现实日映射被列为 P0 开放问题，但它不是参数，是两个核心承诺的直接冲突：

- "对个体产生牵挂"要求 NPC 存活期覆盖多个玩家 Session（首五分钟绑定的是一个具体 NPC）；
- "代际差异驱动回访"要求世代在玩家回访窗口内翻转。

世代按天翻转 → 玩家绑定的 NPC 明天回来已死，每次回访必见讣告；世代按周/月翻转 → 1C 的隔日回访指标在结构上无法归因于进化（测量窗口内进化没发生），四臂在留存指标上被构造性拉平。离线 50 代实验可以任意加速从而回避此问题；线上产品回避不了。应升格为与 DEC-3 同级的正式两难。→ 见提案 P1。

### R2. 文化记忆 → 行为的因果通道未指定

CulturalBelief 是 ≤200 字符自由文本。它影响行为只有两条路：被 LLM 在审议时解读（但审议每天 2–3 次，99% 行为由 Reflex/Utility 决定，Utility 读不了自由文本）；或存在 belief→utility 映射（文档未定义）。结果：代际行为差异几乎必然由 Policy 权重进化主导，文化层大概率是装饰文本。实验可能"通过"（Policy 进化足够），而对外叙事宣称文化传承——这正是第 0 节自己警告的"文案差异冒充代际差异"。→ 见提案 P2 附带项。

### R3. 没有"LLM 无增益"的剧情分支

P0 benchmark 可能显示小模型在结构化候选裁决上不优于效用层最优解（分差小的场景恰是小模型最无优势、且选错代价最小的场景）。若成立：能量定价会把 deliberationEpsilon 推向 0，审议层被进化淘汰；LLM 残余用途只剩文本生成，与"结构化心智，而非全靠提示词"的定位相反。文档只对"多贵"做了止损，没对"有没有用"做止损。→ 见提案 P2。

### R4. 守望两难：提出的补救手段恰好是被禁止的东西

4.1 诊断准确（生效频率可能低到无法支撑情感回路），但补救手段"扩大 epsilon 判定带"就是增加守望的真实权力。受关注血脉获得更多守望 → 注意力与适应度相关 → DEC-2 切断的"注意力驱动进化"从个体层后门回来。需要量化上限，否则 1B 调参会无声越过"表达"与"权力"的界线。→ 见提案 P4 附带项。

### R5. 玩家实际消费的是叙事管线，而它是设计密度最低的子系统

玩家每个 Session 接触的全部是"从模拟真值生成可读叙事"的产物：历史回顾、事件卡、"为什么"因果卡、回访钩子、时刻导演开场。这条管线承担事实准确性（P0 承认"LLM 编造因果"风险）、审核和全部情感表达，却没有 Schema、闸门和评估集。另外：模拟成本 O(NPC)，叙事生成成本 O(玩家)——14.1/14.2 只建模了前者；产品一旦成功，per-player 文本生成会成为主要账单。→ 见提案 P5。

## 二、架构与实验问题

### R6. 审议能量定价（v0.4.1 新增）破坏不变量一

能量定价使"是否发生审议"成为改变世界状态的事件。LLM 超时回退时能量扣不扣？扣，则世界真值取决于机器负载与推理延迟，同 seed 同 Genome 跨机不可复现；不扣，则出现依赖超时的边界策略。无论选哪条，规则必须进权威日志（CanonicalActionEvent 目前无能量扣减字段）。另有四臂混淆：Fixed Utility/Handcrafted 臂不审议、不付能量税，生存闸门的跨臂比较被污染。→ 见提案 P4。

### R7. 预注册的成功判据是析取式的

"至少产生一个稳定、可解释且具有产品意义的增量"——在 4 臂 × 十余指标 × 10–12 seeds 的空间里，"至少一个显著"几乎必然可得（多重比较）。需要主要终点 + 次要终点列表 + 校正方法，且最小效应量必须在见数据前冻结。另：100–170 个成对判断对 60% 偏好率的检验功效约 50–70%，不足 80%。→ 见提案 P3。

### R8. Token 预算算术不自洽，约束量纲选错

6.3 说 25 NPC 全部 LLM token 目标 40k/sim-day；14.1 写"25 NPC × 31k token/sim-day"，按字面矛盾 20 倍。40k ÷（25×2.5 次审议）≈ 640 token/次（含 prompt+completion），现实是 1.5–3k/次，预算大概率一测即破。真正的绑定约束是 调用次数 × 延迟，不是 token；预算与红线应同时按两种量纲计价。

### R9. 时刻导演的供给问题与未命名的治理风险

25 NPC 世界的张力时刻供给速率有限且不受控。队列空虚时最顺手的解法是调世界参数制造戏剧——10.3 只防了世界进化路径，没防运营调参路径。需要纪律：为叙事供给而调整世界参数，同样必须走影子分支（不变量五扩展）。→ 见提案 P5 附带项。

## 三、细节与 Schema 问题

- **R10** `Policy.utilityWeights: z.record(string, number)`：开放字符串键在进化系统里危险——变异能否增删键？父代键集不同时"按字段交叉"如何定义？键集漂移影响哈希与跨代可比性。应收紧为封闭枚举键。
- **R11** DecisionAuditRecord 缺字段：(a) 守望先验是否参与及其内容（UI 宣称"你的守望让它倾向了探索"必须可审计）；(b) 回退原因与能量结算。
- **R12** 候选数不一致：6.2 说 2–5 个候选，Schema 是 max(8)。
- **R13** "保守估计 3:1 以上"（12.3 评估:正典算力比）无任何推导，却被用于经济价值判断。与 v0.4 修掉的"约为单世界方案的 50%"同类。
- **R14** 文档违反自己的第 19 节：仍是单体文档，约六成篇幅是 DEC-3(a) 下可能永远装饰性的条件内容。拆分应是 1A 开工前的第一个交付物，不是"建议"。
