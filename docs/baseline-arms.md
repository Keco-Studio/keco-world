# Four-arm baseline — official run record

**Status:** §17.1 step 3 ("加入 Fixed Utility 与 Handcrafted 基线") — official, single, pre-declared-parameters run. Per the implementation plan (`docs/superpowers/plans/2026-07-22-baseline-arms.md`), this is **the record**: the run command and parameters were frozen in that plan before execution, and — same convention as `docs/degradation-check.md` — the first official run is not rerun with different parameters because a result looks surprising. A collapsing Random arm was the expected sanity-floor outcome, not a failure; it is reported below exactly as it happened, along with one genuinely unexpected result (evolutionary founder intra-diversity, see Findings §4).

**Date:** 2026-07-22
**Commands:** `npm run arms -- run --arm <random|fixed|handcrafted|evolutionary> --seeds 3 --ticks 15000 --chunk 1000`, then `npm run arms -- compare` (default seed `arms-cmp`).
**World:** `makeDemoManifest()` — the same 25-NPC demo world used by `npm run sim` / `npm run evolve` / `npm run degradation`. Per-arm cognition overrides are listed in §1 below.
**Raw output:** `runs/arms/report-{random,fixed,handcrafted,evolutionary}.json` and `runs/arms/compare.json`, generated by these runs, inspected for this document, then deleted per the project's `runs/`-artifacts-not-committed convention (`runs/` is gitignored; verified with `git check-ignore runs/`). Every number reported by the CLI is copied into this document below, so it stands alone without the raw JSON.

## 1. Arm definitions

| Arm | `decisionMode` | `inheritanceMode` | `beliefDynamics` | Roster provenance |
|---|---|---|---|---|
| Random | `random` | `clone` | `off` | `makeDemoRoster(seedRoot)`, `deliberationEpsilon` forced to 0 on every entry |

| Handcrafted | `utility` | `clone` | `off` | `structuredClone(HANDCRAFTED_ARCHETYPES)` — frozen 25-entry hand-authored cast, seed-independent (world layout/founder ages still vary by seed; the cast does not) |
| Evolutionary | `utility` | `breed` | `on` | `makeDemoRoster(seedRoot)` unchanged, including its randomly-varied `deliberationEpsilon` (~20–100 range) |

All four arms carry 25 founders (parity requirement, enforced by `tests/arms.test.ts`).

**跨臂公平性 / designed-epsilon interpretation (frozen, §6.6).** Post-B0 the Resolver's hesitation-band lottery is zero-LLM-cost, so there is no "thinking tax" confound between arms. The remaining fairness question is what `deliberationEpsilon` value each arm is entitled to use. This plan freezes the interpretation as: Random and Fixed use `epsilon=0` (pure/uniform decision, no hesitation band — Fixed is meant to be a clean argmax baseline); Handcrafted uses a **designed, frozen epsilon∈[0,150]** per archetype (part of the hand-authored content, see §2); Evolutionary inherits whatever epsilon the founder roster/mutation produces (the demo roster's ~20–100 range at founding, free to drift under breeding thereafter). This is a content/design choice per arm, not a randomly-drawn nuisance parameter — each arm's epsilon policy is fixed by what that arm is supposed to represent. One material behavioral consequence of this choice was not anticipated when it was frozen and is reported honestly in Findings §4 below: nonzero epsilon turns out to substantially compress behavioral diversity relative to `epsilon=0` on this scenario suite, so "same weights, different epsilon" is not behaviorally neutral.

## 4. Findings

**1. Random arm collapse — sanity floor CONFIRMED, and more severe than "bad policy" would predict.** All 3 seeds went fully extinct (0/25 alive) by generation 2–3, well inside the 15,000-tick horizon. This is not merely a poorly-tuned utility policy: per `src/sim/engine.ts`, `decisionMode: "random"` bypasses the Reflex layer entirely ("no reflex, no utility — uniform over the candidate list" per the code comment) and draws uniformly over the candidate action list, including in the presence of a wolf. No `flee` verb was logged in any random-arm seed at all. The Random arm is therefore a floor on *both* strategic decision-making and basic danger avoidance, which is a stronger (and more honestly stated) sanity check than "random utility weights" would have been.

**2. Fixed vs. evolutionary founder-diversity parity check: NOT at parity, and the gap is large.** The brief calls for checking whether Fixed and Evolutionary founders show comparable intra-arm diversity, since both draw from the identical `makeDemoRoster(seedRoot)` weight distribution. They are not comparable: `intra.fixed = 0.6470` vs. `intra.evolutionary = 0.3396` — evolutionary founders are only **52.5%** as behaviorally diverse as fixed founders on this scenario suite, despite having near-identical underlying `identity`/`policy.utilityWeights`/`policy.thresholds` (same generator, same seed, empty beliefs on both sides). The only structural difference between the two founder sets is `deliberationEpsilon`: Fixed forces it to 0 (pure argmax); Evolutionary keeps the demo roster's randomly-varied ~20–100 range. Tracing the mechanism (`src/mind/resolver.ts`): whenever the top-2 candidate scores fall within `epsilon` of each other, the Resolver replaces the argmax pick with a personality-weighted lottery (`RESOLVER_BASE_WEIGHT=100` dominating a much smaller identity-affinity term). Because that lottery's structure is shared across genomes, genomes that would otherwise diverge on a close call under pure argmax can converge on similar action distributions once epsilon widens the tie-break band. **This is a genuinely unanticipated result of the frozen "designed epsilon" interpretation in §6.6/§1**: nonzero epsilon is not behaviorally neutral — on this suite it roughly halves measured behavioral diversity relative to the same weights at epsilon=0. This matters beyond bookkeeping: it means "founder parity" cannot be assumed just because two arms share a weight-generation process: it also depends on their epsilon policy, and evolutionary's is left to drift, not designed for comparability.

**3. Handcrafted intra-diversity is the highest of any arm.** `intra.handcrafted = 0.6743` > `intra.fixed = intra.random = 0.6470` > `intra.evolutionary = 0.3396`. Despite handcrafted archetypes also carrying nonzero epsilon (0–150, same order of magnitude as evolutionary's ~20–100), the archetypes remain the most behaviorally distinguishable set of the four arms. The most likely explanation is magnitude: handcrafted utility weights, thresholds, and (for 16/25 archetypes) designed belief effects were deliberately calibrated with wide swings (e.g. Sable's `hungerUrgent=420` vs. Quinn's `60`; Tarn's `seekMate=80` vs. Wren's `900`) — larger than the ±150–300 pseudo-random spread `makeDemoRoster` produces — which appears to outweigh the epsilon-driven convergence pressure documented in Finding 2. Read together with Finding 2, the honest picture is: designed content diversity can overcome epsilon-driven behavioral compression, but a randomly-varied roster's diversity is fragile to it.

**4. `fixed | handcrafted` cross distance is unambiguously nonzero: designed rules make Handcrafted behaviorally distinct from Fixed.** `cross[fixed|handcrafted] = 0.6696`, well above 0 and close to the `random|handcrafted` value (0.6696, identical to 4 significant figures — expected, since Random and Fixed founders are literally the same genome content, differing only in the arm's own `decisionMode`, which `compareArms` does not use). This confirms the archetypes' hand-designed utility weights, thresholds, and belief rules do produce a measurably different behavior distribution from a same-size, randomly-varied Fixed roster, not just a different-looking but behaviorally-equivalent cast. As a consistency check on the metric itself: `intra.random == intra.fixed` exactly (0.6470090634441109 in both, to full float precision) because `compareArms` evaluates founder *content* under the scenario suite's own manifest regardless of the arm's `decisionMode` — Random and Fixed literally share the same 25 genomes at `arms-cmp`. `cross[random|fixed] = 0.6211` is consistent with that: of its 625 ordered pairs, 25 are self-pairs at distance 0 (each genome against its identical counterpart) and 600 are between two distinct genomes from the same 25-genome set, so `cross[random|fixed] = 600/625 × intra.fixed = 0.96 × 0.6470 = 0.6211` — matching the reported value exactly. This is offered as a sanity check on the CLI's arithmetic, not a novel finding.

## 5. Implications

**For §17.1 step 9 (formal 50-generation four-arm runs).** All four arms are now implemented (`src/arms/arms.ts`, `src/cli/arms.ts`), so the roadmap blocker on "does a Handcrafted baseline exist" is cleared. This run does not itself satisfy step 9 or §6.7's novelty gates — it is a short (15,000-tick / ~16-generation), 3-seed sanity/calibration run, not the 10–12-seed × 50-generation preregistered formal run §6.7 requires — but it surfaces two things step 9's design should account for: (a) Finding 2 means the formal run's epsilon policy per arm needs to be treated as a first-class experimental design choice, not an afterthought, since it measurably changes founder-level behavioral diversity independent of weight differences; (b) Fixed's tendency to converge to a single surviving lineage that then dominates population growth (`livingLineages=1` in all 3 seeds, highest mean `finalAlive` of any surviving arm at 43.7) suggests Fixed under `clone` inheritance behaves like a coalescent selection process on the initial 25 genomes rather than a static baseline, which is worth flagging for whoever interprets a Fixed-vs-Evolutionary novelty comparison at 50 generations.

**Belief-zeroed control now available for the 1C prereg.** The Evolutionary arm's `beliefDynamics: "off"` cognition variant (already schema-valid and exercised by `makeArmSetup`/`tests/arms.test.ts`) gives a mechanism-isolated control — structural Genome evolution (Identity/Policy breeding + mutation) with cultural-belief inheritance disabled — that was previously a recorded blocker for the §6.4 cultural-layer ablation ("Evolutionary-NoCulture 子臂"). This resolves the *mechanism* half of that blocker: the config exists and runs. The official control comparison (running it as a seed-matched arm alongside the standard Evolutionary arm and measuring the delta) is still the 1C prereg's work, not this task's — no such comparison was run here.
