# 次要终点评分 Rubric

对应 `docs/prereg-1c-draft.md` §5（次要终点，Holm–Bonferroni 家族校正）、§9 交付物清单第 4 项。
本文件冻结三项次要终点的**操作定义**与**评分规程**；评委问卷素材本身（盲化传记）由
`npm run evalpack` 生成，见该文档 §4。三项终点的显著性检验按 Holm–Bonferroni 做家族校正
（`src/analysis/stats.ts` 的 `holmBonferroni`，由 `npm run analyze` 调用），不逐项裸 α=0.05。

## 1. 血脉特征回忆准确率

读完一条传记后，即时作答 **3 道单选题**，每题 4 选项（1 正确 + 3 干扰项，干扰项从同一
传记的其他候选事实/其他血脉传记的对应字段中取，不得瞎编）。题目模板（每条传记按其自身
内容实例化，不是逐字照抄）：

1. **这条血脉最怕什么？**（对应传记中出现频次最高的死因/威胁类型，如"饥饿"、"严冬"、"掠食者"）
2. **这条血脉最兴盛的一代大约是第几代？**（对应 `peakGeneration` 所在代或人口/存活成员数峰值所在代，按最近的十年代分箱：如"第 1–10 代/第 11–20 代/…"）
3. **这条血脉的开创者最看重什么？**（对应传记中渲染出的最高权重 `utilityWeights` 维度，如"觅食"、"庇护"、"探索"、"繁育"）

**计分**：每题 1 分，满分 3 分；报告"3 题全对率"与"平均得分/3"两个数字。正确答案随传记
一起以内部素材形式保留（不进盲化 packet.html，不给评委看），由问卷系统或人工核对判分——
和 `answer-key.json`（评审用哪一臂）是两份独立的密钥表，不得合并存放，避免核对回忆题时
意外向评委暴露臂归属。

## 2. 三代因果复述完整度（0–3 锚定量表）

评委读完传记后，用自由文本回答："请用自己的话简述这条血脉从开创到现在发生了什么，以及
一件事如何导致了另一件事。"由两名评分员（双盲：评分员不知道文本来自哪一臂）独立按下表
打分，每人对**全部**回收的复述文本评分（不分摊、不抽样）：

| 分数 | 判据 | 例句（评分员参考，非评分员唯一可用句式） |
|---|---|---|
| **0** | 无因果——只罗列事实或情绪反应，没有一件事导致另一件事的表述 | "这条血脉活了很多代，中间死了几个人，最后还活着。" |
| **1** | 单代事实——提到至少一个具体因果关系，但只在同一代内，没有跨代传递 | "第 3 代因为一次严冬饥饿死了，饿死之前它一直在囤积浆果。" |
| **2** | 两代因果链——因果关系跨了恰好两代（如上一代的选择/死亡影响了下一代） | "开创者太执着囤粮，很少繁育，导致第 2 代差点断代；第 2 代吸取教训才多生了孩子。" |
| **3** | 三代（或以上）完整链条——因果关系连续跨三代或以上，构成一条可复述的链 | "开创者重囤粮轻繁育，第 2 代差点断代；第 2 代转向重繁育，第 3 代人口暴涨又赶上严冬缺粮，第 4 代因此改回重囤粮——一路能看出这条血脉在囤粮和繁育之间来回调整。" |

**一致性协议**：两名评分员先各自独立评完全部样本，再计算 Cohen's κ
（`src/analysis/stats.ts` 的 `cohenKappa`，两名评分员的 0–3 分向量按样本顺序对齐后传入）。

- **κ ≥ 0.6**：接受，取两名评分员的**平均分**（不取众数、不取较高分）作为该样本的最终得分。
- **κ < 0.6**：不接受当前一致性。先做**逐条分歧样本的会议裁定**（两名评分员当面对每条分歧
  样本达成一致分数），再回看分歧集中在哪个分数边界（通常是 1/2 边界，"跨代"判断最容易分
  歧），**修订本 rubric 的判据措辞或例句**使边界更清晰，然后对**全部**样本（不只是分歧样
  本）重新评分一轮，重新计算 κ。允许迭代，但每次迭代都要在报告里记录 κ 值和本轮修订了什么。

## 3. 关注转化率

读完传记后，单一是/否问题，措辞冻结不得改写：

> **"愿意订阅这条血脉的后续吗？"**（是 / 否）

**计分**：是=1，否=0；报告"是"占比。不设置"不确定"选项——强制二选一，与主终点的"甲/乙"
强制二选一保持同一问卷设计哲学（`docs/prereg-1c-draft.md` §4）。

## 与主分析脚本的关系

`npm run analyze`（`src/cli/analyze.ts`）目前只消费**主终点**判断数据（`--judgments` CSV）
计算 §4 的统计量；本文件三项次要终点的评分数据（回忆答题、因果复述 0–3 分、转化是/否）由
评委问卷系统单独回收，尚未接入 `analyze` 的自动聚合——运行前把三项终点的 p 值列表交给
`holmBonferroni`（`src/analysis/stats.ts`）做家族校正即可复用同一统计基础设施；接入
`analyze` 的自动化留待需要正式跑一整轮次要终点时再做，不在本任务范围内。
