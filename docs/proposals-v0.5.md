## P1. 节奏两难 → 双时标产品结构（回应 R1）

### 判断

牵挂与进化不在同一时间尺度上可感，就不要假装它们在。把产品显式拆成两个时标层，各自有独立的内容供给与留存假设：

| 时标 | 内容 | 留存假设 | 依赖进化？ |
|---|---|---|---|
| **日尺度（个体层）** | 一个 NPC 的困境、选择、关系、生死 | 隔日回访由个体生活事件驱动 | **否** |
| **周/月尺度（血脉层）** | 代际策略变化、文化漂移、家族史 | 周回访由"血脉传记"更新驱动 | 是 |

诚实的推论：**隔日留存不是进化系统的功劳，也不该由它背书。** 1C 若用隔日回访给进化闸门投票，是在测量一个进化按构造无法影响的窗口。

### 参数锚点（待 P0 实测校准，先定初值以便算成本）

- 1 现实日 ≈ 4 模拟日（夜间降速到 2，符合 14.1 的成本阀）；
- NPC 目标寿命 ≈ 60–90 模拟日 → 现实 2–3 周，覆盖 4–8 个 Session；
- 世代间隔 ≈ 现实 1–2 周 → 玩家关注一条血脉一个月，能看到 2–4 次代际更替。

首五分钟绑定的个体在玩家活跃期内大概率存活；其死亡发生在关系已建立、且血脉已有可关注的下一代之后。

### 进化可感性的测量与实时留存解耦

1C 不再指望实时四臂留存差异。改用**压缩血脉传记测试**：

- 从四臂的离线 50 代实验中，按预注册规则自动抽取血脉历史，渲染为等长、盲化的"血脉传记"（时间轴 + 关键事件 + 策略变化摘要，模板生成，见 P5 的接地规则）；
- 人评做成对判断：更想继续看哪条血脉？哪条的三代变化能被复述？
- 实时产品测试（1B）只验证个体层牵挂回路；血脉层可感性由传记测试背书。

### 被否决的替代

- **快世代 + 从一开始只绑定血脉**：抽象对象的情感绑定太弱，首五分钟无法成立；
- **玩家可看加速影子世界**：引入第二套世界状态与叙事口径，1B 不该背这个复杂度。

### 主文档修改点

- 第 18 节 P0 中"模拟日映射"移出，升为 **DEC-4：双时标结构**，含上表；
- 6.7 可观察性闸门改写：隔日回访归个体层假设（1B 验证），血脉传记成对判断归进化假设（1C 主要终点候选，见 P3）；
- 4.4 增补：个体死亡的预期节奏（每关注周期 1–2 次），纪念系统按此频率设计，不按"罕见重大事件"设计。

## P2. LLM null-result 分支（回应 R3，附 R2）

### 预注册的判定基准

P0 benchmark 必须预先定义"LLM 裁决有增益"的可判定标准，建议：

- **测量方式**：在固定情境集的审议触发点上，对"LLM 选择"与"效用层最优"各做 K 条影子 rollout（平台侧，确定性种子），比较 H tick 后的结果指标（生存裕度、目标达成、关系资产）；
- **增益标准**：LLM 选择在 ≥55% 的分歧情境中结果更优（CI 排除 50%），且单次审议成本 ≤ 预算；
- **在 LLM 与效用层同选时不计入**——分差小的情境里两者常同选，只有分歧点才是 LLM 的存在理由。

### 三个预写好的分支

- **B+（有增益、成本可行）**：维持现设计。
- **B0（无增益）**：审议层从阶段一范围移除，替换为"人格加权随机决胜"（epsilon 带内按 Identity 特征做确定性加权抽签——保留个性化的犹豫感，零 LLM 成本，完全可重放）。LLM 收缩为表现层（voice、事件文本、繁殖润色）。**对外定位改写**："结构化心智"的卖点从"AI 裁决"移到"进化的结构化人格 + AI 叙事"。四臂实验照常进行——进化闸门本来就不依赖 LLM 裁决。
- **B±（有增益但超预算）**：审议只保留在出生、死亡边缘、首次接触三类事件（约每 NPC 每周 1–2 次），其余走 B0 机制。

关键点：**阶段一的科学问题（进化产生可感差异）与 LLM 裁决价值是正交的**，null result 不触发 Stop，只触发定位改写。这句话应写进 6.1。

### 附带：修复 R2（文化记忆的行为耦合）

CulturalBelief 增加结构化耦合字段，使信念不经 LLM 也能影响日常行为：

```
effect: {
  target: enum(utilityWeightKey | thresholdKey),   // 封闭键集，见 P4 附带的 R10 修复
  modifier: number,                                  // 有界，如 [-0.3, +0.3]
  condition: TriggerTag | null                       // 如 "winter" | "stranger_present"
}
```

`proposition` 文本降级为该效果的人类可读描述（表现层）。信念的继承、误传与衰减机制不变，但现在误传会真实改变行为——"祖母认为陌生人危险"直接压低 socialTrust 相关权重，而非等 LLM 哪天想起来。这使文化层成为可测的行为因子，四臂实验可以做"文化层消融"（Evolutionary-NoCulture 子臂，seed 允许时）。

### 主文档修改点

- 18 节 P0 第一条扩写为上述判定基准与三分支；
- 6.2 Deliberative 段增加 B0 回退设计；
- 16 节 CulturalBelief Schema 增加 effect 字段。

## P3. 1C 统计方案：主要终点与功效（回应 R7）

### 终点结构

- **主要终点（唯一）**：压缩血脉传记的盲化成对偏好——Evolutionary vs Handcrafted，"更想继续看哪条血脉"。
- **次要终点**（Holm–Bonferroni 校正）：血脉特征回忆准确率；三代因果复述完整度；关注转化率；（个体层，仅描述性）隔日回访。
- **硬闸门（非统计比较）**：生存稳定性、行为新颖性，按预注册阈值判定，不参与终点族。

### 功效分析

成对偏好为二项检验，H0: p=0.5，α=0.05 双侧：

| 真实偏好率 | 80% 功效所需判断数 |
|---|---|
| 0.60 | ≈ 195 |
| 0.62 | ≈ 135 |
| 0.65 | ≈ 85 |

原文"100–170 个独立成对判断"对 60% 偏好率功效仅 50–70%。**决定**：最小产品意义效应量定为 0.62（低于此的偏好差异撑不起产品差异化主张），判断数下限提为 **150，目标 200**；每人最多贡献 8 个判断且素材不重叠，聚类稳健标准误复核。若预算不支持 200 判断，收窄素材多样性而不是降 n。

### 预注册纪律

冻结于首个正式 seed 启动前：终点定义、效应量、n、抽取传记的自动规则（禁止人工挑选素材——这是 15 节自己列的"人评素材挑选偏差"威胁的执行答案）、排除标准、分析代码 commit。Iterate 决策后若改指标，新一轮实验必须换新 seed 组。

### 主文档修改点

6.7 全段替换为上述结构；15 节阶段一威胁"人评素材挑选偏差"绑定"自动抽取规则 + 预注册"为缓解措施。

## P4. 审议能量结算的确定性规则 + 四臂去混淆（回应 R6，附 R4、R10、R11）

### 结算规则：触发时计费（Rule B）

能量在**审议触发时**扣除，与 LLM 是否按时返回无关：

- 触发条件 = f(效用分差, deliberationEpsilon, 世界状态)——全部确定性输入，跨机可复现；
- 触发后：LLM 成功 → actionSource=deliberative；超时/非法 → actionSource=fallback。两者能量成本相同（"想了但没想明白也是想了"）；
- CanonicalActionEvent 增加 `deliberationTriggered: boolean`，能量扣减由它确定性推导，重放层一闭合。

残余非确定性只剩"选了哪个动作"（LLM vs 回退），这本来就由行动日志承载、由审计层管辖——不变量一的边界恢复干净。回退率作为仪表指标监控，实验中回退率 >10% 视为基础设施故障，该 seed 作废重跑，而不是让污染数据进闸门。

### 四臂去混淆：能量定价移出离线实验

能量定价是**线上世界的经济平衡机制**，不是离线实验需要的变量。1A/1C 四臂中关闭能量定价，审议成本控制仅靠硬上限（4 次/日）。理由：

- 消除 Evolutionary 独付"思考税"的生存混淆；
- 硬上限已阻断"epsilon 拉满"的进化磁吸（v0.4.1 引入定价的原始动机在离线场景不成立）；
- "深思血脉 vs 节能血脉"的进化分支是线上长期观赏性设计，留待 1B 后在活世界中灰度启用并单独观察。

这部分收回 v0.4.1 补丁的适用范围，属有意为之，应在变更日志中明示。

### 附带：守望权力上限（R4）

守望影响加两条量化红线，写进不变量四附近：

1. 守望介入的决策占比 ≤ 5%（按血脉滚动 30 日窗口统计）；
2. 定期对照审计：镜像 Genome 在无守望影子分支中重跑抽样区段，存活率差异超过预注册界限时自动降低该血脉守望权重并告警。

epsilon 判定带的任何扩大都必须在这两条红线内。若 1B 实测"生效频率不足以支撑情感回路"且红线不允许再扩，结论是**调整表达形式**（如守望影响梦境/反思文本等纯表现层输出），而不是继续加权力。

### 附带：Schema 补丁（R10–R12）

- `utilityWeights`/`thresholds` 改为封闭枚举键的 object，键集随 WorldManifest 版本化；交叉定义为逐键交叉，变异不得增删键；
- DecisionAuditRecord 增加：`patronInfluence: { theme, weightDelta } | null`、`fallbackReason: enum | null`、`energyCharged: number`；
- candidateActions 上限统一为 5（与 6.2 一致），Schema max(5)。

## P5. Chronicle：叙事子系统一等公民化（回应 R5，附 R9、R8）

### 定位

Chronicle 是体验层内的独立子系统，负责一切"从模拟真值到玩家可读文本"的转换：历史回顾、事件卡、"为什么"卡、回访钩子、血脉传记、墓志上下文。它与时刻导演并列：导演选"看什么"，Chronicle 决定"怎么讲"。

### 接地规则（防"LLM 编造因果"）

1. **模板优先**：每类 SemanticEvent 有确定性模板渲染路径；LLM 只做润色，且润色输出与模板输出同源事件集；
2. **因果只来自模拟**：死亡链、决策候选、效用分数从权威日志与审计记录读取渲染；LLM 不得推断未记录的因果（"为什么"卡直接展示审计记录的候选与分数摘要）；
3. **可溯源 Schema**：`ChronicleCard { cardType, eventIds[], auditRefs[], renderedText, renderPath: template|llm_polished, groundingVersion }`——每张卡的每个事实主张能回指事件 ID；
4. **幻觉审计集**：每周抽样 N 张卡人工核对主张与日志，主张级准确率目标 ≥99%；跌破即回退纯模板模式（安全降级不阻塞 tick，与 13.5 同款纪律）。

### 成本结构：从 O(玩家) 压回 O(血脉)

- 回顾与传记按 `(lineageId, timeInterval)` 生成并缓存，同血脉的所有关注者共享；
- 守望个性化（"你的守望让它倾向了探索"）是确定性模板叠加层，读 DecisionAuditRecord.patronInfluence（P4 补丁），零额外 LLM 成本；
- 14.2 增加第四项情景变量：Chronicle 缓存命中率。冷门血脉长尾用纯模板兜底。

### 时刻导演治理（R9）

- 新增仪表指标 **张力供给率**（单位现实小时内新入队的合格开场时刻数），随 1B 上线监控；
- 不变量五扩展一句：**为叙事供给而调整世界参数（灾害频率、资源紧缩等）等同于世界规则变更，必须走影子分支晋升**——运营手不能伸进活世界找戏；
- 供给不足的合法手段按序：扩大可开场时刻的判定口径 → 增加 NPC 数量（世界规则变更，走流程）→ 接受"安静时段"并以历史回放填充开场。

### 预算量纲（R8）

- 统一数字：删除 14.1 的"31k"，以 6.3 的 40k/60k 为准，并注明其为**待实测占位**；
- 预算双量纲计价：token/sim-day 与 **审议调用数 × p95 延迟**；50 代实验的墙钟预估按调用数模型给出（25 NPC × 2.5 次/日 × 世代长度 × 50 代 × seeds ÷ 并行度 × p95），benchmark 报告必须填这张表。

### 主文档修改点

- 第 5 节体验层增补 Chronicle 段（与时刻导演并列）；
- 17 节 1B 交付物增加"Chronicle v0（模板路径 + 幻觉审计集）"；
- 14 节按上述修改成本模型；
- 18 节 P0"关注摘要如何避免 LLM 编造因果"移除——由接地规则回答，不再是开放问题。
