## 0. 执行摘要与边界
Living Worlds 的首要产品承诺不是“把玩家电脑变成算力网络”，而是：
> **让玩家在十分钟内对一个并非预写剧情驱动、会成长、繁衍和消亡的 AI 血脉产生牵挂，并愿意回来查看它后来发生了什么。**
因此，项目的第一性验证顺序是：
1. NPC 的代际差异是否真实存在，而不是文案差异；
2. 差异是否能被普通玩家看懂、记住并复述；
3. 玩家是否愿意关注、回访和为历史留下痕迹；
4. 在上述条件成立后，再验证跨世界、分布式算力、训练和世界进化。
以下内容明确不属于阶段一目标：
- 不证明浏览器算力网络可以替代云计算；
- 不建设可兑换、可交易或带现金价值的经济系统；
- 不要求 LLM 跨硬件位级重现；
- 不训练基础模型或血脉 LoRA；
- 不让线上世界自动根据即时热度改写规则；
- 不把“世界很复杂”当作“游戏很好玩”的替代证据。
## 1. 一句话概述
Living Worlds 是一个持续运行的 AI 生命观察游戏：NPC 在确定性世界规则中生活、形成关系、繁衍与消亡；其人格、策略和文化记忆在代际中变化。玩家不直接操控 NPC，而是关注血脉、理解事件、留下命名与纪念，并在有限、公平的边界内影响其思考方向。跨世界迁徙用于检验策略泛化；分布式算力、模型训练和世界进化只在核心观看与回访价值被证明后逐步启用。
## 2. 为什么做这个
### 2.1 产品机会
现有 AI NPC 项目常见两种形态：一类是昂贵但短暂的技术演示，NPC 能对话，却缺少长期后果；另一类是把 LLM 接入传统任务系统，角色说得更多，但世界本身没有真正变化。Living Worlds 的机会在于把 AI 能力放进一个具备死亡、继承和历史连续性的系统，使玩家看到的不是“角色随机聊天”，而是“某种生活方式为何在这个世界里延续或消失”。
### 2.2 三个核心差异
**死亡与选择。** NPC 会因饥饿、寒冷、风险判断和社会关系而失败，幸存者留下后代。选择压力提供方向性，但阶段一不会假设“生存得好就一定好看”，因此必须同时验证行为新颖性和可观察性。
**进化的结构化人格 + AI 叙事，而非"AI 裁决"（v0.5.1/B0 改写）。** NPC 的反射、效用决策、人格、策略和文化记忆均有明确数据结构；行为差异完全由可进化的结构化心智产生，文化信念通过 effect 字段直接作用于效用层（v0.5/P2）。LLM 不参与权威决策——benchmark 证明小模型在结构化候选裁决上系统性差于随机（DEC-5）；它的职责是把结构化生命翻译成可读的故事：Chronicle 叙事、voice、事件文本与繁殖润色。卖点是"这些差异是真的、且讲得出来"，不是"AI 在替它思考"。
**玩家拥有情感关系，而非角色控制权。** 玩家可以关注血脉、查看因果、记录历史、表达希望其关注的方向，但不能像传统 RTS 一样发号施令。早期设计遵循“表达优先、权力后置”：先证明玩家在乎，再讨论算力是否应产生生存优势。守望影响受量化红线约束（v0.5/P4，见 4.1）。
### 2.3 与 Ironveil 的关系
对 Ironveil 而言，Living Worlds 可以成为三个方向的验证场：
- 证明结构化游戏数据、模拟协议和 AI 行为工具能够支撑长期运行的内容；
- 为创作者提供可测试的 NPC、世界规则与行为评估接口；
- 在核心产品成立后，为批量游测、影子模拟和模型评估提供真实任务需求。
但它不应被包装为“反低质量 AI 内容”的口号工程。旗舰内容的说服力来自可玩的结果，而不是系统复杂度或技术叙事。
### 2.4 明确不是什么
Living Worlds 不是加密挖矿产品，不是完全去中心化的共识系统，不是纯粹的人工生命研究沙盒，也不是依靠无限生成文本维持新鲜感的聊天房间。浏览器算力是可选参与机制；权威世界状态、产品体验和安全责任仍由平台承担。
## 3. 系统不变量与架构决策
任何新机制必须先检查以下不变量。它们比具体模型、运行时和经济参数更稳定。
### 3.1 不变量一：双层重放
所有权威声明分为两层。
**第一层：世界模拟重放（权威真值）。** 给定世界 manifest、初始 roster、确定性种子、外部输入序列和结构化行动日志，参考模拟器必须重放出相同状态轨迹与结果。JSON 使用固定规范化方案后哈希；按固定 tick 间隔保存 checkpoint 哈希；分歧可定位到首个不一致 tick。
**第二层：决策审计（来源与合规）。** 每个进入行动日志的 NPC 决策保留观察、候选动作、Genome 快照、模型与运行时版本、提示模板、原始结构化输出、合法性结果和成本指标。该层要求可追溯、可复核、可界定漂移，不要求跨硬件位级一致。
世界真值由第一层决定；“这个动作是否真的由指定心智产生”由第二层评估。两者不得混为一谈。
**认知开销结算属于第一层（v0.5/P4；v0.5.1 后休眠但原则保留）。** 任何改变世界状态的认知开销必须由确定性输入推导——这是原则，比具体机制长寿。B0 后 Resolver 纯代码零成本，此条款无实际计费对象；若 prereg v2 重开任何有成本的认知机制，按此原则接入（触发即计费、`deliberationTriggered` 入权威日志、扣减确定性推导），不得重新发明。
### 3.2 不变量二：Schema 约束接口
Observation、Action、Genome、WorldManifest、任务载荷、行动日志、重放声明和审核结果均为版本化结构化数据，并在跨边界时校验。LLM 只能在明确候选动作和输出 Schema 内生成结果。文本展示与权威行为分离：文本漂移不得改变世界真值。玩家可见叙事必须走 Chronicle 接地管线（v0.5/P5，见第 5 节），每个事实主张可回指事件 ID。
### 3.3 不变量三：权威状态不在任意浏览器间分片
线上世界的权威状态由平台参考协调器或受控执行集推进。浏览器与 BYOC 节点只接收：
- 独立情境决策；
- 从签名 checkpoint 启动的有限长度 rollout；
- 反思、摘要、审核、嵌入和批量评估；
- 可独立验证的 TestScenario。
不得把同一连续世界的任意 region/tick 范围直接分给不可信节点共同修改。状态化分片会引入跨区依赖、顺序一致性、检查点传输和回滚问题，超出早期产品所需。
### 3.4 不变量四：单向、无现金价值的参与经济，与表达/权力边界
算力与参与奖励只能用于游戏内装饰、历史记录权限、体验便利和确定性贡献等级；不可兑换现金、不可交易、不可与创作者分成打通。未成年人可参与的版本不提供预测押注、随机兑奖或近似博彩机制。
**守望权力红线（v0.5/P4）。** 玩家对 NPC 的任何影响机制必须同时满足：
1. 守望介入的决策占比 ≤ 5%（按血脉滚动 30 日窗口统计）；
2. 定期对照审计：镜像 Genome 在无守望影子分支中重跑抽样区段，存活率差异超过预注册界限时自动降低该血脉守望权重并告警。
任何影响参数（含 epsilon 判定带）的调整都必须在这两条红线内进行。红线存在的原因：受关注血脉获得更多守望，若守望产生生存优势，注意力就会与适应度相关——DEC-2 从世界层切断的"注意力驱动进化"会从个体层回来。
### 3.5 不变量五：慢层变更必须经影子分支晋升
NPC 行为可实时变化；Genome 按代变化；世界参数变化更慢。任何影响线上世界气候、拓扑或规则的候选变更，必须先在影子分支中运行生态测试、玩家冲击测试和回滚检查，再按纪元发布。线上注意力只能成为候选排序信号，不能直接驱动实时变异。
**叙事供给同受此约束（v0.5/P5）。** 为叙事供给而调整世界参数（灾害频率、资源紧缩等）等同于世界规则变更，必须走影子分支晋升——运营手不能伸进活世界找戏。时刻导演队列空虚的合法处置见第 5 节。

### DEC-1A［已决］：世界模拟真值
采用双层重放。阶段一、二不要求跨硬件复现 LLM token，只要求相同结构化行动日志产生相同世界结果。
### DEC-1B［待决，阻塞阶段三高价值经济声明］：决策来源真实性
仅验证行动日志自洽，无法证明行动确实由指定 Genome 和模型产生。作弊节点可能手写合法最优动作。阶段三开工前必须选择组合方案：
- 服务器在随机 tick 发出不可预知的决策挑战，并要求先提交承诺再揭示完整审计记录；
- 高价值远征在参考节点或两个独立节点上冗余执行；
- 对抽样决策做候选集、效用分布和选择一致性复核；
- 将高价值奖励限制在平台可验证执行路径；浏览器路径只获得低价值贡献奖励。
在该问题解决前，不应把远征结果称为“可无信任定价”。
### DEC-2［已决，v0.4］：在线世界适应度
孵化期和在线期不直接共享单一适应度。孵化期使用生态硬门槛、行为新颖性和内测评审；在线期使用延迟、去刷后的玩家信号对影子候选排序。胜出候选仍需生态门槛、回归测试和人工发布批准。实时线上世界不自行根据注意力突变。
### DEC-3［待决，产品身份，无阻塞但需在阶段三前确认］
v0.4 的三项修正（算力网络降为可选增益、供养降为表达型、孵化正典收回平台）叠加后，本项目的实际定位已从"分布式 LLM 游戏网络"移向"平台运营的 AI 生命观察游戏，附带可选社区算力参与"。每步修正各有充分理由，但方向漂移应被有意识地确认而非默认接受。两个选项：**（a）接受当前定位**——算力是风味与参与机制，不是骨架；经济叙事、先驱者凭证与对外表述全部按此校准。**（b）恢复算力的结构性地位**——则须正面解决不变量三与孵化经济的矛盾（受信执行集的扩容路径、BYOC 节点的信任分级、正典推进权的下放条件），代价是重新引入 v0.4 刻意排除的工程复杂度。此为产品身份判断，非技术判断；在阶段三（首个涉及外部算力经济的阶段）开工前必须拍板，当前默认按（a）执行。
### DEC-5［已决，2026-07-21，B0］：审议无增益，LLM 退出权威决策
预注册 benchmark（docs/bench-prereg-v1.md）官方运行结果：qwen3 三尺寸分歧胜率 42–44%、CI 上界全部 < 0.50，且全部低于随机非最优对照（53.9%）——有统计把握的负增益。依 §18 P0 预写规则触发 **B0**：审议层从阶段一移除，替换为人格加权确定性决胜（Resolver，见 6.2）；LLM 收缩为表现层；四臂实验不受影响（进化闸门与裁决价值正交）。完整决策记录与随机对照异常分析见 docs/decisions/DEC-P0-deliberation-B0.md。效力边界：结论只约束 Phase 0 世界的结构化候选裁决；Phase 1 引入社交/繁殖后如需重测，须发布 prereg v2。**遗留审计项（P1 前）**：随机对照 >50% 说明 epsilon 带内效用排序对长期结果几乎无信息量，效用权重的长期预测力需单独审计。
### DEC-4［已决，v0.5/P1］：双时标产品结构
牵挂与进化不在同一时间尺度上可感，产品不假装它们在。显式拆成两个时标层，各自有独立的内容供给与留存假设：

| 时标 | 内容 | 留存假设 | 依赖进化？ |
|---|---|---|---|
| **日尺度（个体层）** | 一个 NPC 的困境、选择、关系、生死 | 隔日回访由个体生活事件驱动 | **否** |
| **周/月尺度（血脉层）** | 代际策略变化、文化漂移、家族史 | 周回访由"血脉传记"更新驱动 | 是 |

诚实的推论：**隔日留存不是进化系统的功劳，也不该由它背书。** 1C 不用隔日回访给进化闸门投票——那是在测量一个进化按构造无法影响的窗口。个体层牵挂回路由 1B 实时验证；血脉层进化可感性由压缩血脉传记盲测背书（见 6.7）。
**节奏参数锚点（待 P0 实测校准，先定初值以便算成本）：**
- 1 现实日 ≈ 4 模拟日（夜间降速到 2，符合 14.1 成本阀）；
- NPC 目标寿命 ≈ 60–90 模拟日 → 现实 2–3 周，覆盖 4–8 个 Session；
- 世代间隔 ≈ 现实 1–2 周 → 关注一条血脉一个月，能看到 2–4 次代际更替。
首五分钟绑定的个体在玩家活跃期内大概率存活；其死亡发生在关系已建立、且血脉已有可关注的下一代之后。
**被否决的替代**：快世代 + 从一开始只绑定血脉（抽象对象的情感绑定太弱，首五分钟无法成立）；玩家可看加速影子世界（引入第二套世界状态与叙事口径，1B 不该背这个复杂度）。
## 4. 玩家体验与旅程
产品体验必须先于进化、算力和训练叙事。玩家第一天只需要理解三件事：这些 NPC 有自己的生活；他们可能失去机会或死亡；我可以选择在乎谁，但不能直接控制他。
### 4.1 三种玩家身份
**观察者（Observer）。** 零投入进入。第一屏同时展示“一个可读的个体”和“仍在运行的村落”：镜头默认跟随一名正面临具体问题的 NPC，背景中能看见其他居民和资源活动。玩家可切换俯瞰、个体跟随和事件时间线。
**守望者（Patron，替代早期“算力供养者”定义）。** 玩家关注一条血脉，并选择一个有限影响主题，例如探索、建立关系、储备或建造。该主题只在 Resolver 犹豫带内成为轻量权重倾斜（v0.5.1/B0：挂接点从审议层移到决胜层），不越过反射与生存规则，不直接增加资源、生命或繁殖概率。核心感受是“我表达了期待，但它仍然自己做决定”。
**可见性机制（v0.4.1 引入，v0.5/P4 修订，v0.5.1 更新）。** 守望主题只在犹豫带触发时生效。B0 的一个副产品对此有利：Resolver 零成本，犹豫带触发频率不再受审议预算（每日 2–3 次）压制——epsilon=60 校准实测显示犹豫场景远多于原预算假设，守望的可感知生效机会随之增加；权力上限仍由不变量四的两条红线（介入决策占比 ≤5%、无守望影子对照审计）封顶，可见性问题的压力从"生效太少"移向"红线内选择哪些生效时刻做标注"。规则不变：当守望主题实际参与了某次决胜的倾斜，对应事件卡必须明确标注（如“它犹豫时，你的守望让它倾向了探索”），且该宣称可回溯到审计记录的 `patronInfluence` 字段；历史摘要中累计展示守望生效次数与走向。**“守望生效频率 × 玩家感知满意度”仍列为 1B 阶段的正式测试假设。** 若红线不允许扩大影响，结论是**调整表达形式**（梦境、反思文本等纯表现层输出），而不是继续加权力。（v0.5.2 注：决胜层倾斜已实现——PATRON_TILT=150，patronInfluence/影子对照审计与红线度量见 docs/product-shell.md）
**先驱者（Pioneer）。** 通过持续观看、有效反馈、测试贡献和可选算力贡献获得确定性贡献等级。权益以新世界内测、历史档案权限、纪念物样式和创建观察任务为主，不承诺选址优势或资源优势，避免早期形成付费/硬件阶层。
### 4.2 首个五分钟
**0:00–0:20：先看到一个问题，不先看教程。** 玩家进入时镜头跟随一名 NPC。界面只显示名字、血脉、当前目标和一句可理解的风险，例如“冬天还有两天，他的储备只够一天”。背景中村落持续运行。
**0:20–1:00：建立因果。** 玩家看到 NPC 在两个合理行动之间选择，并能展开“为什么”卡片：当前需要、人格倾向、已知经验和候选动作。系统不展示完整提示词，只展示可读的因果摘要（由 Chronicle 从审计记录渲染，不允许 LLM 推断，见第 5 节）。
**1:00–2:00：第一次表达。** 玩家选择“关注这个血脉”，并从探索、关系、储备、建造中选择一个守望主题。界面明确提示：这不是命令，只会在它犹豫时形成轻微影响。
**2:00–4:00：看到后果或悬念。** NPC 执行行动，出现一个可观察的短期结果；若长期结果尚未发生，系统把它加入“接下来值得看”的事件卡。玩家可点开父母、子代或近期关键事件，理解血脉不是单个聊天角色。
**4:00–5:00：形成回访理由。** 系统生成一句不剧透的回访钩子，例如“第一场寒潮后，这个决定会接受检验”，并允许玩家订阅站内提醒。此时才简短解释：世界不会因你离线暂停，角色会成长、繁衍，也可能消亡。
首个五分钟不展示算力、代币、模型、锦标赛、LoRA 或世界进化术语。
### 4.3 典型 Session 循环
一次常规 Session 目标时长为 5–12 分钟：
1. **历史回顾（必看，30–60 秒）**：展示玩家离线期间与关注血脉直接相关的 1–3 个高影响事件，附因果摘要；若期间守望主题曾生效，明确标注生效时刻与走向；
2. **检查关系与状态（必看，1–2 分钟）**：查看存活成员、出生、迁徙、关系变化和未解决风险；
3. **表达一次意图（可选，30 秒）**：调整守望主题、创建观察任务、命名事件或留下纪念；
4. **观看实时片段（可选，2–8 分钟）**：跟随个体、俯瞰村落或回放关键事件；
5. **留下回访锚点（必看，10 秒）**：系统列出下一次可能发生的重要窗口，但不保证结果。
“必看”部分负责让玩家快速恢复上下文；“可选深入”部分负责满足愿意研究系统的玩家。任何 Session 都不要求玩家保持标签页开启才能维持关注血脉的正常生存。
### 4.4 死亡、失败与纪念
死亡必须具有后果，但不能通过细节刺激或惩罚玩家。界面展示原因链、遗留影响、后代和历史位置，不展示夸张画面。玩家可建立墓志、保存一段事件回放、给未命名事件命名。系统不得暗示“玩家少开了一会儿标签页导致它死了”。
**死亡节奏预期（v0.5/P1）。** 按 DEC-4 的节奏锚点，玩家每个关注周期（约一个月）预期经历 1–2 次所关注个体的死亡——死亡是常规节拍，不是罕见重大事件。纪念系统按此频率设计：轻量、快速、可累积（墓志与家族史自然衔接），而非每次都要求玩家进行沉重的仪式性操作。
### 4.5 纪元事件
低风险气候变化以季节预报和历史年表表现。拓扑和规则变化只通过影子分支晋升，并在正式发布前经历三个阶段：预兆、公开说明、纪元切换。玩家关注的血脉不得被无预警规则变更瞬间清除；高风险变化必须提供回滚或迁徙方案。
## 5. 架构总览
系统按职责分为七层。
**体验层（Client Experience）。** Excalibur.js 负责渲染、镜头、时间线、个体卡片和事件回放。用户界面消费结构化事件，不直接读取模型内部状态。前台本地模型可以生成对话表面文本和摘要，但这些文本默认不改变权威世界状态。
**时刻导演（Moment Director，v0.4.1）：** 首个五分钟假设"进入时总有一个 NPC 正面临可读的问题"——这不会自然发生。导演服务持续扫描模拟中的张力时刻（储备将尽、关系破裂、远行未归、抉择临近），按可读性与悬念打分维护"可开场时刻"队列；新玩家进入或回访时从队列分配镜头。导演只选择观看对象与镜头语言，不干预模拟本身。**供给治理（v0.5/P5）：** 新增仪表指标**张力供给率**（单位现实小时内新入队的合格开场时刻数），随 1B 上线监控。供给不足的合法手段按序：扩大可开场时刻的判定口径 → 增加 NPC 数量（世界规则变更，走影子分支）→ 接受"安静时段"并以历史回放填充开场。为戏剧调整世界参数必须走影子分支（不变量五）。
**Chronicle 叙事子系统（v0.5/P5，与时刻导演并列）：** 负责一切"从模拟真值到玩家可读文本"的转换——历史回顾、事件卡、"为什么"卡、回访钩子、血脉传记、墓志上下文。导演选"看什么"，Chronicle 决定"怎么讲"。四条接地规则：
1. **模板优先**：每类 SemanticEvent 有确定性模板渲染路径；LLM 只做润色，且润色输出与模板输出同源事件集；
2. **因果只来自模拟**：死亡链、决策候选、效用分数从权威日志与审计记录读取渲染；LLM 不得推断未记录的因果（"为什么"卡直接展示审计记录的候选与分数摘要）；
3. **可溯源 Schema**：ChronicleCard 的每个事实主张能回指事件 ID（见第 16 节）；
4. **幻觉审计集**：每周抽样 N 张卡人工核对主张与日志，主张级准确率目标 ≥99%；跌破即回退纯模板模式（安全降级不阻塞 tick，与 13.5 同款纪律）。
**成本结构**：回顾与传记按 `(lineageId, timeInterval)` 生成并缓存，同血脉的所有关注者共享——叙事成本为 O(血脉)，不是 O(玩家)；守望个性化（"你的守望让它倾向了探索"）是读 `patronInfluence` 字段的确定性模板叠加层，零额外 LLM 成本。
**权威模拟层（Canonical Simulation）。** 平台参考协调器推进世界 tick、执行合法动作、更新资源与关系、写入行动日志和 checkpoint。模拟层不依赖 LLM 可重现；只依赖结构化动作。阶段一可完全单机运行，线上阶段采用受控服务或有限受信执行集。
**认知层（Cognition）。** Reflex、Utility、Resolver 三层决策（v0.5.1/B0：Deliberative 由 Resolver 替代）。代码生成候选动作并评分；候选分差进入 epsilon 犹豫带时，由人格加权确定性决胜裁决——纯代码、零 LLM、完全可重放。LLM 不出现在认知层；语言全部在体验层（Chronicle）生成。
**生命层（Organism）。** NPC 由 Identity、Policy、Cultural Memory 和可选 Neural Adapter 组成。阶段一启用前三层。繁殖的结构继承和变异由代码完成；LLM 只润色非权威文本；入世前经过一致性与内容审核。
**任务与算力层（Job Fabric）。** 浏览器、BYOC 和平台 GPU 通过统一任务协议执行有限、可验证的工作。浏览器不共同维护连续世界状态，而是执行独立决策、有限 rollout、反思、摘要、审核和测试情境。平台 GPU 承担高信任审核与训练。
**参与与经济层（Participation）。** 奖励观看、反馈、测试和可选算力贡献。奖励不产生现金价值，不直接购买生存优势。贡献等级采用确定性门槛和封顶，不使用软通货预测押注。
**发布与进化层（Release & Evolution）。** 世界、模型和规则都以版本化 artifact 发布。任何世界变更先进入影子分支，经过生态、回归、玩家冲击和安全闸门后再晋升到下一纪元。
## 6. 阶段一：单村落产品—系统联合验证
### 6.1 目标与范围
阶段一只证明一件事：
> 在固定世界规则下，进化 NPC 是否能产生可测、可理解、可记忆，并足以驱动回访的代际差异。
范围为一个村落、20–30 个 NPC、一个共享小模型候选（0.5–4B，最终按实测选择）、单机参考环境、固定世界参数、无分布式、无训练、无经济和无权力型供养。世界参数仍按 WorldGenome 存储，但阶段一不变异。
**科学问题与 LLM 裁决价值正交（v0.5/P2；v0.5.1 已应验）。** 阶段一验证的是"结构化 Genome 进化产生可感差异"，四臂实验的进化闸门不依赖 LLM 裁决是否有增益。P0 benchmark 已判定无增益并触发 B0（DEC-5）：审议层已替换为 Resolver、定位已改写，阶段一照常推进——这正是当初预写该分支的目的。
### 6.2 三层行为架构
**Reflex。** 危险规避、受击反应、极低能量、继续已确认路径等。纯代码，每 tick 可检查。
**Utility。** 觅食、储备、避寒、休息、常规社交和返回住所。由 Policy 权重、阈值和世界状态评分。默认在状态显著变化或每 10–30 tick 重新计算。文化信念的 effect 字段直接修饰对应权重与阈值（v0.5/P2）。
**Resolver（v0.5.1/B0，替代 Deliberative）。** 候选分差进入 epsilon 犹豫带（top-2 分差 ≤ epsilon）时触发：按 Identity 特征对带内候选做**人格加权确定性抽签**（权重由 riskTolerance/explorationBias/patience 等映射，随机性来自无状态键控 RNG，key 含 npcId 与 tick）——保留个性化的犹豫感与行为多样性，零 LLM 成本，完全可重放。不同人格在同一犹豫场景下有不同的倾向分布，这本身是可进化、可观察的行为因子。守望主题在此层内做红线内的轻量倾斜（见 4.1）。
**原 Deliberative 设计（休眠，DEC-5）。** "LLM 裁决 2–5 个候选"的设计经预注册 benchmark 判定为负增益，从阶段一移除。审计记录、候选集与 fallback 语义等基础设施保留（benchmark 管线即建于其上）；若 Phase 1 场景（社交谈判、长期计划）重开此问题，须先发布 prereg v2 重测，不得默认恢复。
### 6.3 推理预算与仪表化（v0.5.1/B0 改写为表现层口径）
B0 之后，认知层零 LLM 成本（Resolver 纯代码）。LLM 预算全部落在**表现层**：Chronicle 叙事（回顾/事件卡/传记，按血脉×时段缓存，O(血脉)）、出生润色、voice 文本。纪律不变：
- 表现层 token 预算随 1B Chronicle v0 实测后冻结；原 40k/60k per sim-day 数字作废（其口径是审议调用，已不存在）；
- **预算双量纲计价（v0.5/P5，沿用）**：token 与 调用数 × p95 延迟。官方 benchmark 实测参考：qwen3:0.6b p50 544ms / 4b p50 1497ms（结构化小请求，per-call 开销主导）；
- 记录 prompt/completion tokens、延迟分位、失败率、缓存命中率（Chronicle）、sim-day wall-clock；
- 超标时先降低润色比例、提高模板路径占比和缓存粒度，不减少实验 seed、代数或强对照组；
- 语言全部在事件层生成，权威日志只含结构化动作。语言漂移不能改变动作。
**休眠条款（原 P4 审议计费，随 DEC-5 冻结）。** 触发时计费（Rule B）、审议硬上限（世界规则而非基因组字段）、能量定价线上限定与四臂关闭——这套设计随审议层一并休眠，完整保留于本节 v0.5 版文本与 DEC-5 记录中；若 prereg v2 重开审议，按原样恢复，不重新发明。`deliberationTriggered`/`energyCharged` 字段保留在 Schema 中（恒为 false/0），保证日志向后兼容。
### 6.4 Genome 四层
**Identity。** 风险偏好、社交信任、探索倾向、耐心、表达风格等小型 typed object。低频、小幅变异。
**Policy。** 效用权重、阈值、规则和 Resolver 犹豫带参数（deliberationEpsilon，v0.5.1 起为人格表达度旋钮）。**键集封闭且随 WorldManifest 版本化（v0.5/P4）**：交叉定义为逐键交叉，变异只改值、不得增删键。按字段交叉，中频变异。
**Cultural Memory。** 带置信度、来源和时间衰减的信念。可遗忘、误传和修正；只继承信念子集，不继承完整事件日志。**结构化行为耦合（v0.5/P2）：** 每条信念携带 effect 字段（目标权重/阈值键、有界修饰量、可选触发条件），不经 LLM 直接作用于效用层——"祖母认为陌生人危险"直接压低 socialTrust 相关权重，误传会真实改变行为。proposition 文本降级为 effect 的人类可读描述（表现层）。这使文化层成为可测行为因子，四臂实验可做文化层消融（Evolutionary-NoCulture 子臂，seed 预算允许时）。
**Neural Adapter。** 字段预留，阶段一禁用。阶段四只有在结构化 Genome 已经证明不足时才启用。
### 6.5 繁殖流水线
父母选择 → 结构字段交叉 → 数值与规则变异 → 文化信念抽样 → 代码级约束检查 → LLM 润色 voice/措辞 → 内容与一致性判别 → 冻结并哈希。
结构结果在 LLM 调用前已经成立。LLM 失败时使用模板化文本，不阻断出生。审核拒绝文本不等同于杀死血脉；系统保留安全占位文本并进入复核队列。
### 6.6 四臂对照
- **Random**：仅用于 sanity check；
- **Fixed Utility**：普通游戏 AI 基线；
- **Handcrafted**：人工设计人格、效用差异和有限事件脚本；
- **Evolutionary**：Identity/Policy/Cultural 可遗传与变异。
Handcrafted 组必须记录内容制作工时和规则数量。否则“进化是否减少内容成本”无法公平比较。Fixed Utility 与 Handcrafted 组都应拥有与 Evolutionary 相同数量的可识别角色原型，避免把“有没有个性”与“会不会进化”混在一起。
**跨臂公平性（v0.5/P4，v0.5.1 后自动满足）：** B0 后认知层零成本，"思考税"混淆源已不存在；Resolver 对四臂一视同仁（Fixed Utility/Handcrafted 臂不启用犹豫带或 epsilon=0，按各臂设计冻结）。
（0.5.2 注：Handcrafted 臂采用设计冻结的 epsilon∈[0,150]，理由与记录见 docs/baseline-arms.md）
### 6.7 三轴成功闸门（v0.5/P1+P3 重构）
**生存稳定性（硬约束，非统计比较）。** 每正式实验臂 10–12 个独立 seed、每 seed 50 代。不得出现系统性灭绝、无限人口或资源失控。阈值在预注册实验方案中冻结。
**行为新颖性（硬约束 + 排序，非统计比较）。** 使用 30–50 个固定情境和若干连续情境。测量行动分布距离、序列 n-gram、状态访问分布和可复现涌现事件。比较的不仅是“Evolutionary 内部血脉差异”，还包括其相对同规模 Fixed Utility/Handcrafted 原型的增量。
**可观察性（推进闸门，统计终点结构如下）。**
- **主要终点（唯一）**：压缩血脉传记的盲化成对偏好——Evolutionary vs Handcrafted，"更想继续看哪条血脉"。传记从离线 50 代实验中按**预注册规则自动抽取**（禁止人工挑选素材），由 Chronicle 模板渲染为等长、盲化的时间轴 + 关键事件 + 策略变化摘要。
- **次要终点**（Holm–Bonferroni 校正）：血脉特征回忆准确率；三代因果复述完整度；关注转化率。
- **个体层指标（仅描述性，归 1B 假设）**：隔日主动回访、继续观看时长。按 DEC-4，这些指标不给进化闸门投票。
- **功效分析**：成对偏好为二项检验（H0: p=0.5，α=0.05 双侧）。80% 功效所需判断数：偏好率 0.60 → ≈195；0.62 → ≈135；0.65 → ≈85。**最小产品意义效应量定为 0.62**（低于此的偏好差异撑不起产品差异化主张），判断数下限 **150、目标 200**；每人最多贡献 8 个判断且素材不重叠，聚类稳健标准误复核。若预算不支持 200 判断，收窄素材多样性而不是降 n。形成性测试仍可用 12–20 人。
- **预注册纪律**：终点定义、效应量、n、传记自动抽取规则、排除标准、分析代码 commit，全部冻结于首个正式 seed 启动前。Iterate 决策后若改指标，新一轮实验必须换新 seed 组。
阶段一只有在主要终点显著且方向正确、硬闸门全部通过时通过。仅优于 Random 不算通过。
### 6.8 阶段一交付物
1. 仪表化与基准报告（含调用数×延迟墙钟模型）；
2. 可重放模拟内核（**已交付**，阶段 0）；
3. 三层行为系统（Reflex/Utility/Resolver，v0.5.1/B0）；
4. Genome 与繁殖管线（含文化 effect 耦合）；
5. 固定情境测试集、行为度量与**血脉传记自动抽取器**；
6. 四臂离线实验；
7. 首个五分钟可点击/可玩壳；
8. 人评与回访报告；
9. Go / Iterate / Stop 决策记录。
## 7. 阶段二：多世界与泛化验证
阶段二的目标不是“再做一张地图”，而是验证 World Protocol 能否支持同一心智在未见环境中工作。
### 7.1 协议
核心 schema 包括 Observation、Action、WorldManifest、SemanticEvent 和 CapabilitySet。世界差异通过资源、规则和能力声明表达，不允许 NPC 依赖特定世界的自然语言名字。协议从 v0 开始版本化，并规定：
- 必填字段的向后兼容；
- 新动作和资源类型的能力协商；
- 未知字段的忽略/拒绝策略；
- 世界版本与测试情境集的绑定关系。
### 7.2 泛化实验
至少准备三个训练/开发世界和两个完全保留的 held-out 世界。比较：
- Random；
- Fixed Utility；
- Handcrafted；
- 只在单一世界进化的 Genome；
- 在多个开发世界轮换进化的 Genome。
关键指标不是单纯存活率，而是**泛化差距**：开发世界表现减去 held-out 世界表现。只有当进化 Genome 在 held-out 世界上相对 Fixed Utility/Handcrafted 仍保有优势，且差距可解释时，才证明获得可迁移策略。
若只在第二世界与 Random 比较，会重复阶段一的弱对照问题，也无法区分“记住某类资源布局”和“获得一般策略”。
## 8. 阶段三：远征与跨世界声望
阶段三把泛化结果转化为玩家可理解的远征、声望和历史事件，但不默认建立无信任经济。
### 8.1 远征
默认使用副本制：血脉生成有版本号的远征快照，本体继续留在原世界。失败损失远征资源和声望进度，不删除本体。永久性“真朝圣”只在产品测试证明玩家理解风险后作为少量高戏剧性玩法出现。
目的地由平台在兼容世界池中随机或分层随机分配。随机分配能降低共谋，但不是唯一防线；还需世界版本冻结、创作者关联检测、重复配对降权和奖励上限。
### 8.2 世界难度评级：不用简单 Elo
“远征存活率的反函数”不是 Elo。世界难度与 Genome 能力是非对称、带版本、可能多维的关系。早期采用：
- 世界版本的经验存活率与置信区间；
- 按气候、资源稀缺、敌对风险等标签分层；
- 新世界使用保守先验和低奖励上限；
- 数据充分后再使用分层 Logistic/IRT 模型，估计 Genome 能力、世界难度和交互项。
任何评级都必须绑定 `worldVersionHash`。世界规则更新后不能沿用旧难度分数。
### 8.3 声明与决策来源
远征声明包含权威模拟重放和决策审计。低价值社区远征可以接受审计抽样；高价值声望、排行榜或先驱权益必须走平台参考执行或冗余执行。
建议的反伪造流程：
1. 服务器签发远征 seed、checkpoint 和决策挑战计划承诺；
2. 节点按 tick 提交行动与审计记录承诺；
3. 远征结束后服务器随机揭示需完整复核的 tick；
4. 节点提交候选动作、Genome、模型和运行时证明；
5. 参考复核判断 legal、legal-but-drift 或 inconsistent；
6. 只有合规声明进入高价值评级。
在 DEC-1B 完成前，文档应使用“可审计声望”而不是“可无信任定价”。
## 9. 阶段四：数据与模型改进管线
浏览器不做梯度训练。分布式部分是经验生成和评估，梯度计算由平台 learner 完成。
### 9.1 推荐顺序
**路径一：离线行为克隆 / SFT。** （前提：prereg v2 重开审议且判定有增益，否则本路径无数据源）从高质量、已重放、去重的审议记录中学习“情境 + 候选动作 → 选择与理由”。这是第一条主路径，因为目标清晰、可离线评估、易回滚。
**路径二：偏好与排序学习。** 使用人工或规则构建的成对选择，训练候选排序器或轻量 critic。优先训练独立选择器，而不是立即改动共享基础模型。
**路径三：血脉适配器。** 只有当结构化 Identity/Policy/Cultural 无法维持稳定人格或决策风格时，才引入 LoRA。适配器绑定基座哈希和评估报告，不能成为不可解释的默认遗传层。
**路径四：强化学习。** 仅在奖励定义、离线评估和回滚成熟后研究。生存奖励容易产生保守、刷分或伤害可观察性的策略，不能作为第一条训练路线。
**ES 对高维 LoRA 的扰动** 保留为研究分支，不列为主交付路径。其样本效率、噪声和浏览器前向成本需要单独实验，不应假设“世界天然就是有效的高维适应度预言机”。
### 9.2 数据闸门
训练样本必须满足：
- 世界模拟重放通过；
- 决策来源达到相应信任级别；
- 相似轨迹与相同行动激进去重；
- 不把高声望等同于高质量，需结合 held-out 情境表现；
- 内容审核与隐私同意通过；
- 记录数据来源、模型版本、世界版本和删除状态。
### 9.3 评估与发布
候选模型必须在冻结测试集、held-out 世界、人格一致性、安全和成本基准上比较。部署采用影子运行和小比例纪元测试。任何新模型都必须能一键回滚到旧基座与旧适配器组合。
“4B LoRA 单卡过夜”只能作为待实测假设。训练时长取决于序列长度、样本量、量化/精度和硬件，不能写成架构事实。
### 9.4 许可与数据权利
阶段四前冻结选定基座许可证、微调后再分发条件、模型命名要求和可接受使用政策。玩家轨迹训练采用单独、明确同意；未成年人数据默认不进入训练集；删除请求至少能阻止未来训练并从可定位数据集移除。
## 10. 阶段五：世界进化与纪元发布
世界进化不是线上参数自动漂移，而是**候选世界变更的生成、影子评估和版本晋升**。定性说死（v0.4.1）：**线上世界不进化；进化只发生在影子分支，线上世界只接收经全部闸门晋升的版本。**任何实现若让线上参数绕过影子流程直接变异，即违反不变量五。
### 10.1 世界 Genome
世界 Genome 分为三类：
- **气候与资源参数**：刷新率、季节曲线、灾害频率与强度；
- **拓扑参数**：地图生成、区域连通、新区域与资源点；
- **规则参数**：交易、再生、繁殖和交互规则。
三类变更的最小发布间隔和允许幅度依次降低。高风险规则变更必须由显式 migration 脚本处理旧存档与现有血脉。
### 10.2 影子分支流程
1. 从当前线上版本复制多个影子分支；
2. 生成一个受约束的世界变更；
3. 使用历史玩家轨迹、合成玩家和代表性 Genome 运行长程模拟；
4. 检查种群边界、多样性、资源循环、可达性、必胜策略和迁移安全；
5. 通过生态门槛后，进行内测与可观察性评审；
6. 使用延迟、去刷后的玩家信号对候选排序；
7. 低风险变更可自动进入发布候选，高风险变更需人工批准；
8. 以纪元版本发布，并保留旧版本与回滚点。
### 10.3 适应度结构
**生态健康是硬门槛。** 灭绝、无限增长、不可达资源、行为塌缩和单一必胜策略均直接淘汰。
**行为新颖性是第二道门槛。** 世界变更应产生新的适应问题，而不是只提高数值难度。
**玩家价值是排序信号。** 只使用去刷、延迟和聚合后的指标，例如关注转化、事件复述、隔日回访和观看完成率。即时停留时长或算力贡献不能直接成为世界适应度，以免系统进化出操纵注意力的奇观。
### 10.4 速率分离
| 层级 | 变化对象 | 最小尺度 | 发布方式 |
|---|---|---|---|
| 行为 | 单次动作 | tick | 实时 |
| Genome | 人格/策略/文化 | 代 | 出生时冻结 |
| 气候资源 | 世界低风险参数 | 数十代 | 影子评估后小版本 |
| 拓扑 | 连通与新区域 | 纪元 | 预告后纪元版本 |
| 规则 | 基础交互规则 | 多纪元 | migration + 人工批准 |
速率分离降低共同进化震荡，但不能单独证明稳定。仍需影子长程模拟、变异幅度上限、冻结开关和回滚。
### 10.5 成功判据
至少在多 seed、跨一个拓扑变更的长程实验中满足：
- 无系统性震荡或生态崩溃；
- 至少一次世界变化引起可复现、可解释的策略 meta 转移；
- 观察者能识别时代变化，并认为其增强而非破坏长期关系；
- 纪元发布后关键留存和关注指标无不可接受回归；
- 回滚演练通过。
## 11. 孵化管线与候选世界锦标赛
### 11.1 目的与成本归属
候选孵化用于寻找更好的初始条件、世界参数和历史背景，不是为了制造“完全保密”的假象。浏览器节点能看到其任务数据，因此安全目标是最小披露、难以拼接、篡改可发现和泄漏可追责。
**成本归属（v0.4.1 明确）。** 依不变量三，候选世界的正典演化时间线只能由平台或受控执行集推进——这是孵化的主计算成本，落在平台侧。浏览器与 BYOC 承担的是评估性工作：从签名 checkpoint 出发的分支 rollout（多个分支中只有平台选定的一条并入正典，其余作为探索与评估样本）、固定情境批次、行为度量与游测。因此“玩家为看不见的大陆供能”在字面上只对评估算力成立；先驱者凭证奖励的是评估贡献与参与，而非正典演化本身。经济叙事与凭证定价必须按此校准，不得暗示浏览器贡献是孵化的存续前提。
### 11.2 Successive Halving
以 8 个候选、4 个等长阶段为例，如果单个世界完整成熟成本为 `C`，则总计算约为：
`8×C/4 + 4×C/4 + 2×C/4 + 1×C/4 = 3.75C`
因此，该方案约为“完整成熟全部 8 个候选”成本的 47%，但约为“只成熟一个世界”成本的 3.75 倍。原“约为单世界方案的 50%”表述错误，应删除。
是否值得支付 3–4 倍探索成本，取决于候选质量差异和最终发布频率。阶段一、二不建设该管线；阶段三以后先以 4→2→1 小规模验证。
### 11.3 评审
每轮评审按顺序执行：
1. 生态与协议硬门槛；
2. 行为新颖性；
3. 成本与稳定性；
4. 盲评可观察性；
5. 设计者复核异常案例。
人类评审可以触发复评，但不单方面推翻硬门槛。commit-reveal 只证明决定在某时刻已提交，不能证明决定本身公平，因此仍需公开评审规则和版本化指标。
### 11.4 败者内容
淘汰世界可以保留为研究档案或非正史传说，但“把败者全部迁入胜者世界”不是默认操作。大量外来血脉可能破坏胜者生态和叙事清晰度。只允许少量、经过兼容测试的代表性血脉迁移，并明确其来源。
### 11.5 温室问题
孵化后期注入历史玩家轨迹、合成玩家和小规模真实内测。上线初期不得临时大幅提高变异率来“快速适应”，因为这会让发布闸门失去意义。更安全的做法是：
- 预留上线冲击影子分支；
- 使用资源调节和事件频率等可回滚参数；
- 必要时暂停新出生或远征，而非加速整个基因库变异。
## 12. 算力网络与参与经济
### 12.1 产品立场
算力网络是可选增益，不是世界存活、NPC 在线或商业模型成立的前提。玩家关闭标签页后，权威世界继续正常运行；平台不得通过“不开机就会饿死你关注的 NPC”制造压力。
### 12.2 适合下放的任务
优先级从高到低：
1. 固定情境的结构化决策批次；
2. 从签名 checkpoint 启动的有限 rollout；
3. 反思、摘要、翻译和低风险文本审核；
4. TestScenario 游测与回归；
5. 嵌入和资产标注；
6. 高信任任务的冗余副本。
不下放：
- 同一线上世界的连续权威 tick；
- 需要跨 region 实时同步的任务；
- 训练梯度；
- 未脱敏的高敏剧情与玩家数据；
- 无法独立验证且会产生高价值奖励的开放式任务。
### 12.3 任务单位
任务应是不可变输入到不可变输出：
`Task = signedCheckpoint + boundedHorizon + policySnapshot + seed + capabilitySet`
节点先提交结果哈希，再按抽样要求揭示行动与审计记录。状态变化只在平台验证后合并为实验结果；浏览器结果不直接写入线上权威世界。**分支 rollout 的正典化：**同一 checkpoint 可分发给多个节点跑不同分支；平台验证后至多选择一条并入正典时间线，其余分支保留为评估数据（行为分布采样、反事实比较、canary 校验），不视为浪费——但成本核算时须承认评估算力与正典算力的比例（**待实测占位，v0.5 注：此前"保守估计 3:1 以上"无推导依据，实际比例须在孵化管线小规模验证中测量**），这决定了浏览器贡献的真实经济价值。
### 12.4 奖励
贡献奖励使用确定性等级、每日封顶和次线性曲线。奖励可包括：
- 装饰与纪念物样式；
- 世界内测资格；
- 历史档案和高级观察工具；
- 贡献徽章；
- 非竞争性的排队便利。
删除“候选世界胜者预测押注”。即使只使用软通货，该机制仍会引入赌博式预期、未成年人风险和刷号动机，与核心体验无关。
算力贡献不能提高单一血脉的生命值、资源刷新或基础生存概率。未来若测试“更多思考预算”，必须在所有玩家都有基础额度、影响封顶且与硬件性能脱钩的条件下进行，例如平台将贡献折算为同额的有限思考预算券（若彼时存在有成本的认知机制），而不是按原始 GPU 吞吐直接加权。
### 12.5 同意与资源保护
必须显式 opt-in，显示当前任务、预计资源占用、累计贡献和暂停按钮。使用电池、页面后台、设备过热、帧时间恶化或网络受限时自动暂停。移动端默认禁用。模型下载大小、缓存占用和删除入口也必须透明。
## 13. 内容与行为安全管线
安全分为两类，不能只审核文本。
### 13.1 行为安全
权威 Action Schema 限制可执行动作；世界规则检查目标、距离、资源和冷却。阶段一不提供开放式工具调用、任意代码执行或现实世界外部操作。攻击、交易和繁殖等高影响动作必须有明确游戏规则和年龄分级边界。
### 13.2 文本安全
触发点包括：
- 出生时的身份/文化文本；
- 玩家可见对话与摘要；
- 世界发布前的历史档案；
- 训练数据入集前。
流程为确定性规则 → 小型分类器/LLM 语境审核 → 高影响对象人审。中英混合内容必须覆盖。每个审核结果绑定规则集、模型版本和语言。
### 13.3 误杀与回退
审核失败不直接删除血脉。结构化 Genome 保留，玩家可见文本替换为安全模板并进入复核。若问题来自结构化目标或行动倾向，而不是措辞，则冻结该个体的相关高风险动作并迁移到安全策略，不得只换文案。
### 13.4 未成年人和玩家输入
玩家命名、墓志和观察任务均走单独审核。未成年人默认不能创建公开自由文本，只能使用受控模板或经过审核的命名。玩家举报、申诉和内容屏蔽必须有可执行流程。
### 13.5 成本与可用性
普通对话不必在生成前使用最昂贵模型逐条审核，可采用本地规则、轻量分类器、生成后阻断和抽样复核。发布历史、训练数据和高关注血脉使用更严格门槛。安全降级不能阻塞权威世界 tick。
## 14. 成本模型
“平台侧边际推理成本下降”比“边际成本趋近零”更准确。成本包括：
- 本地/平台推理的电力和硬件折旧；
- 模型与更新包的 CDN 流量；
- 浏览器模型缓存与用户存储；
- 权威模拟与协调服务；
- 行动日志、checkpoint、审计记录和事件回放存储；
- 冗余验证和参考复核；
- 审核、人评和客户支持；
- learner GPU 与模型评估；
- 合规、删除请求和安全运营；
- **Chronicle 叙事生成与缓存（v0.5/P5）**。
### 14.1 阶段一
25 NPC 的初始预算以 40k token/sim-day 为目标、60k 为红线（**待实测占位**；v0.4.1 中"31k"数字来历不明，已删除），且必须在真实模型上测量。**预算同时按 token 和 调用数×p95 延迟 双量纲计价（见 6.3）。** 成本报告至少包含：
- 每 sim-day wall-clock（按调用数模型预估并实测校验）；
- 每代总 token、总调用数和总时长；
- 叙事/润色/审核占比（v0.5.1：审议项随 B0 移除）；
- 回退率；
- 每 50 代实验的电量估算；
- 四臂实验与人评的人力成本。
若 50 代无法在可接受墙钟内完成，优先减少 LLM 参与范围或采用更小模型，而不是把长程模拟偷偷改成少量演示片段。
**常驻世界成本（v0.4.1 补，v0.5 更新）。** 自 1B 形成性测试起，"世界不因你离线暂停"的承诺意味着平台须 24/7 运行一个含 LLM 调用的活世界——这是项目第一笔持续月度账单，与离线实验成本性质不同。量级按 DEC-4 节奏锚点计算：25 NPC × 4 模拟日/现实日（夜间降速 2）× 表现层 token 预算（1B Chronicle v0 实测后冻结，见 6.3——B0 后模拟本体零 LLM 成本，账单主体是 Chronicle 生成），加协调与存储。夜间降速运行是合法成本阀（低活跃时段降低 tick 率与叙事生成频率，只要不停摆）。**叙事成本结构（v0.5/P5）：** 模拟成本 O(NPC)，Chronicle 按 `(lineageId, timeInterval)` 缓存共享，成本 O(血脉)；守望个性化为零 LLM 成本的模板叠加。冷门血脉长尾用纯模板兜底。
### 14.2 线上阶段
平台成本不一定“主要在存储与协调”。模型分发、参考执行、审核和冗余验证可能成为主要项，取决于参与率和模型大小。必须用三种情景建模：
- **低贡献率**：大部分工作由平台承担；
- **中贡献率**：浏览器覆盖批量评估；
- **高贡献率**：协调、验证、CDN 和反作弊成本上升。
每种情景增加第四个变量：**Chronicle 缓存命中率（v0.5/P5）**——它决定叙事成本随玩家数的实际增长曲线。
经济模型必须在低贡献率情景下仍能运行，否则产品对浏览器政策和用户硬件过度依赖。
## 15. 威胁模型
### 阶段一：产品与研究风险
- 生存最优但观看无聊；
- 手工对照过弱或过强；
- 行为度量选择错误；
- 人评素材挑选偏差——**缓解已绑定（v0.5/P3）：传记素材按预注册规则自动抽取，禁止人工挑选；分析代码 commit 冻结**；
- LLM 成本使实验无法完成；
- 结构化变异过硬导致角色机械，过松导致不可解释；
- **时刻导演供给不足诱发运营调参（v0.5/P5）——缓解：张力供给率仪表 + 不变量五扩展（为戏剧调参必须走影子分支）**；
- **Chronicle 幻觉损害因果信任（v0.5/P5）——缓解：接地规则 + 每周幻觉审计集，跌破 99% 回退纯模板**。
### 阶段二：协议与泛化风险
- Schema 过窄限制世界，过宽失去可验证性；
- held-out 世界泄漏；
- 资源/地形标签导致隐性过拟合；
- 测试情境集随版本漂移，历史结果不可比。
### 阶段三：声明与经济风险
- 女巫账号和贡献刷取；
- 创作者关联世界共谋；
- 世界版本更新后沿用旧难度；
- 行动日志自洽但决策来源伪造；
- 参考复核带宽不足；
- 声望被误解为现金价值或可交易资产。
### 阶段四：训练风险
- 数据投毒和伪造高质量轨迹；
- 合成数据闭环导致能力坍缩；
- 高生存样本压制社交与可观察性；
- 许可或再分发不合规；
- 玩家删除请求无法追踪；
- 新基座改变旧血脉身份。
### 阶段五：世界进化风险
- 共同进化震荡；
- 世界向当前强势血脉谄媚；
- 注意力目标诱发操纵性奇观；
- 变更破坏旧血脉和玩家承诺；
- 影子测试不覆盖真实上线行为；
- migration 或回滚失败。
### 分布式算力风险
- 浏览器后台节流、WebGPU 覆盖和模型下载阻碍参与；
- 节点返回懒惰、复制或伪造结果；
- 任务数据泄漏和水印误判；
- 高性能硬件形成隐性权力；
- 用户误解资源消耗；
- 任务状态过大导致网络成本高于节省的推理成本。
### 内容、隐私与平台风险
- 演化文本跨代漂移；
- 玩家自由文本污染世界与训练集；
- 未成年人内容与数据处理；
- PIPEDA、Québec Law 25、GDPR 等同意与删除要求；
- 平台服务中断造成“活世界”承诺破裂；
- 第三方模型或浏览器政策变化。
每项威胁必须绑定：检测信号、缓解措施、负责人、阶段闸门和残余风险。叙述性威胁列表不能替代运行手册。

- 任务最小化，只发送完成该 rollout 所需数据；
- 专有名词替换为带语法属性的占位符；
